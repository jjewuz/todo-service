# 📝 Сервис To-Do на React

Менеджер задач с возможностью добавления, удаления, копирования и редактирования дел. Все данные автоматически сохраняются в браузере (localStorage) и загружаются при следующем визите.

## 🚀 Демо

> [Ссылка на GitHub Pages](https://jjewuz.github.io/todo-service/)

## 🛠️ Технологии

- **React** (компонентный подход, JSX, хуки useState, useEffect)
- **localStorage** (автосохранение списка задач)
- **CSS** (верстка по БЭМ-методологии)

## 📋 Функциональность

### 1. Общая структура

- Приложение полностью реализовано на React
- Интерфейс собран из JSX (без ручных DOM-операций)
- Проект запускается без ошибок, верстка не сломана

### 2. Компонентная архитектура

- Логика организована через React-компоненты
- Компоненты имеют понятные названия (App, TaskItem, TaskForm и т.д.)
- Компоненты разделены по ответственности
- Повторяющаяся разметка задачи вынесена в отдельный компонент TaskItem

### 3. Управление состоянием

- Список задач хранится в состоянии React (useState)
- Исходный список (defaultItems) не изменяется напрямую
- Все обновления списка идут через создание нового массива (иммутабельность)

### 4. Отображение задач

- При загрузке отображаются задачи из localStorage или предустановленный список по умолчанию
- Список рендерится через map()
- Для каждого элемента используется корректный уникальный key (не индекс, а id)

### 5. Добавление задачи

- Присутствует форма добавления
- Поле ввода — контролируемый компонент (связан с состоянием)
- Новая задача добавляется в **начало** списка
- Пустые задачи не добавляются (.trim())
- После отправки форма очищается

### 6. Удаление задачи

- У каждой задачи есть кнопка удаления
- По нажатию задача удаляется из состояния и пропадает со страницы

### 7. Копирование задачи

- Есть кнопка копирования
- Создается полная копия задачи
- Копия добавляется в начало списка

### 8. Редактирование задачи

- Кнопка редактирования меняет режим отображения на инпут
- Текст задачи можно изменить
- Редактирование завершается по Enter или кнопке сохранения
- Отмена (Escape или кнопка) возвращает исходный текст
- Редактирование не ломает разметку и не влияет на другие задачи

### 9. Работа с localStorage

- При первой загрузке данные читаются из localStorage
- Если сохраненных задач нет — используется defaultItems
- При любом изменении списка (добавление, удаление, копирование, редактирование) данные автоматически сохраняются в localStorage

### 10. Хорошие практики

- Безопасный вывод текста (без dangerouslySetInnerHTML)
- В коде нет console.log, alert, debugger
- Имена переменных и функций — осмысленные, в camelCase
- Отсутствуют неиспользуемые переменные и лишний код
- Код логично организован и легко читается
- Компоненты не перегружены: логика вынесена в отдельные функции/хуки

## 📦 Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git

# Перейти в папку проекта
cd todo-service

# Установить зависимости
npm install

# Запустить в режиме разработки
npm run dev

# Деплой на GitHub Pages
npm run deploy
```
